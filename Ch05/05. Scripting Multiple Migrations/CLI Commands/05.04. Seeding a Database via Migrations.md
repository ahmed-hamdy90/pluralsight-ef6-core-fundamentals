## Follow Steps to Build Seeds Data and use EF 6 Migration For Run them

***NOTE:*** During Course She works to display Steps using Windows, Visual Studio and PMC. The next Steps for using CLI Commands (Using Linux) with Dotnet CLI Or Use new Dockerfile for Development

#### Create new Migration file To Read new changes for Seeding Data

1. Run EF tool Command Direct from The Root Project with mention about Data Project `PublisherData`

```SHELL
dotnet ef migrations add seedauthors --project=PublisherData/
```
#### Run EF migrations update command on Local Or use New Development Docker

1. Make Sure the new Previous Database details which created by EF Migration

2. Run EF Update command 

    - Locally

    ```SHELL
    dotnet ef database update --project PublisherData/
    ```

    - Through Docker

    ```SHELL
    docker compose -f docker-compose.development.yaml build
    docker compose -f docker-compose.development.yaml up
    docker compose -f docker-compose.development.yaml exec ef5coreconsole bash
    ```
    Inside Container 

    ```SHELL
    dotnet ef database update --project PublisherData/
    ```

#### ***Optional, Not Done through Course*** Create new Migration SQL file includes Seeding Data

```SHELL
dotnet ef migrations script --project PublisherData/ --output PublisherData/GeneratedSQL.20251015205929_seedauthors.sql
```
