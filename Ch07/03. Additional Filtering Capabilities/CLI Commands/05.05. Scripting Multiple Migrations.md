## Follow Steps to Create EF 6 Migration New SQL file After Adding Seeding Data

***NOTE:*** During Course She works to display Steps using Windows, Visual Studio and PMC. The next Steps for using CLI Commands (Using Linux) with Dotnet CLI Or Use new Dockerfile for Development

#### Create SQL file using Idempotent option To Make Difference between Migrations files

```SHELL
dotnet ef migrations script --project PublisherData/ --idempotent --output PublisherData/GeneratedSQL.Idempotent.sql
```

#### Create SQL file using From Parameter

This Generated file will include the second Changes for Migrations files Only which include Seeding Data and discard the Initial Migrations files

```SHELL
dotnet ef migrations script Intial --project PublisherData/  --output PublisherData/GeneratedSQL.FromInitial.sql
```

#### Create SQL file using From And To Parameters

This Generated file will include the Initial Migrations files and Discrad the Second Migrations file which include Seeding Data

```SHELL
dotnet ef migrations script 0 Intial --project PublisherData/  --output PublisherData/GeneratedSQL.WithoutSeedAuthors.sql
```
